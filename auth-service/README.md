# В Путь! — Сервис Авторизации (`auth-service`)

Микросервис аутентификации и авторизации для платформы «В Путь!». Отвечает за управление пользователями, сессиями, ролями и безопасную выдачу токенов.

## 🛠 Технологический стек

* **Язык:** Python 3.13
* **Менеджер пакетов:** [uv](https://github.com/astral-sh/uv) (современная и быстрая альтернатива pip/poetry)
* **Фреймворк:** FastAPI / Uvicorn
* **ORM:** SQLAlchemy 2.0
* **Миграции:** Alembic
* **База данных:** PostgreSQL 15
* **Контейнеризация:** Docker / Docker Compose

---

## 🔐 Архитектура безопасности

В сервисе реализована гибридная схема авторизации на базе JWT токенов:

1. **Access Token (Stateless):** Короткоживущий токен. Проверяется другими микросервисами асинхронно без обращения к базе данных `auth-service`.
2. **Refresh Token (Stateful + Ротация):** Долгоживущий токен. Хранится в базе данных **исключительно в виде криптографического хэша (SHA-256)**, что исключает компрометацию сессий при утечке БД. При каждом обновлении токена происходит его ротация (выпуск новой пары), защищающая от Replay-атак.
3. **Управление сессиями:** Модель сессий поддерживает привязку к `IP` и `User-Agent` для возможности реализации фичи «Выйти со всех устройств».

## Быстрый старт (Локальная разработка)

### Предварительные требования
Убедись, что у тебя установлены `Docker`, `Docker Compose` и локально установлен `uv`.

### 1. Клонирование и настройка окружения
Создай файл `.env` в папке `auth-service/environment/` по образцу из `.env.example`

### 2. Запуск через Docker Compose
Из корня всего проекта (где лежит общий `docker-compose.yml`) запусти сборку и контейнеры:
```bash
docker compose down
docker compose up --build
```
* Бэкенд будет доступен по адресу: `http://localhost:8001`
* Документация Swagger: `http://localhost:8001/docs ` (пока нет)

## ⚗️ Работа с миграциями (Alembic)

Поскольку в `docker-compose.yml` настроен изолированный анонимный том для виртуального окружения, команды Alembic внутри контейнера нужно запускать через `uv run`.

### Применение миграций на базу данных
```bash
docker exec -it auth uv run alembic upgrade head
```

### Откат последней миграции
```bash
docker exec -it auth uv run alembic downgrade -1
```

## 📝 Основные сущности БД (Схема)

* **`Role`** — Ролевая модель пользователей (`id`, `title`, `created_at`).
* **`User`** — Данные пользователя (`id`, `login`, `hashed_password`, `is_active`, `is_verified`). Пароль опционален для будущей поддержки OAuth2.
* **`UserSession`** — Сессии и refresh-токены (`id`, `user_id`, `refresh_token_hash`, `expires_at`, `user_agent`, `ip_address`). Связана каскадным удалением с пользователем.
